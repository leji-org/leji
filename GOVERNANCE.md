# Governance

Leji is an open specification. Its goal is to be a neutral, vendor-agnostic standard for the shared context layer of AI-native teams.

- **Creator and steward.** Leji was created by Vuong Nguyen. Meteor Dreams, LLC is the current steward: it maintains the specification and this repository, and reviews proposals.
- **Neutrality.** The specification isn't tied to any tool, model vendor, or consultancy. Conformance never requires a commercial product or service.
- **Stewardship is transferable.** The steward role is designed to be transferable to a neutral foundation or multi-party group if adoption warrants it. The permissive licenses and this document exist so that transfer changes nothing for adopters.
- **Changes.** The specification changes by proposal: open an issue describing the problem and the intent, then a pull request against `spec/`. Normative changes require a changelog entry and a version bump per the spec's own versioning rules.
- **Decision rule.** The steward decides and records the decision; dissenting proposals stay open and documented rather than closed. As external maintainers join, decision-making moves to a named maintainer group with a defined review quorum; until then the steward is the sole maintainer.
