# Porting TypeScript → Go / Python

The three reference SDKs implement the same `leji` CLI and must stay at parity (see
[System Invariants](../system/invariants.md)). TypeScript at `packages/sdk` is the source of truth;
`packages/sdk-go` and `packages/sdk-py` mirror it. A behavior change lands in all three and ships at
one coordinated version. This is the discipline for getting a port byte-identical the first time.

## The rule: mirror, don't reimplement

- **Node is the reference. Port the behavior, not your idea of it.** Read the TypeScript, reproduce
  what it does, and resist refactoring mid-port; a "cleaner" Go or Python shape is how divergence
  enters. Improvements go back into the Node reference first, then port forward.
- **Parity is byte-identical, not behaviorally-equivalent.** `npm run parity` (`scripts/parity-test.ts`)
  runs all three CLIs on identical inputs and asserts identical **stdout, stderr, exit code, and written
  file tree** (bytes **+ mode + symlinks**). The only field allowed to differ is the context-index
  `generatedAt` timestamp, which the harness normalizes field-aware. The viewer/serve output is not
  byte-compared; everything else is. "Looks the same" is not the bar; the bytes are the bar.

## Divergences that bite

Standard libraries disagree on details that parity catches. Match the Node reference, not the local idiom.

- **HTML escaping.** Node escapes `'` as `&#39;` and `"` as `&quot;`. Python's `html.escape` emits
  `&#x27;`, Go's `html.EscapeString` emits `&#34;`. Don't use the stdlib escaper; use a hand-written
  helper that reproduces Node's five replacements in order (`&` `<` `>` `"` `'`). See
  `packages/sdk-py/.../viewer_cmd.py` `_html_escape` and its Go counterpart, both of which carry a comment
  saying exactly this.
- **Never reserialize structured data to edit it.** Go's `encoding/json` alphabetizes map keys and drops
  unknown fields; Python preserves insertion order. Round-tripping `leji.json` through a
  parse→mutate→serialize cycle reorders keys and silently discards anything the struct doesn't model;
  instant parity failure. Edit the **raw text in place** instead, the way the Node manifest splices do
  (`bindAgentInManifestText` / `declareVendorAdapterInManifestText` in `packages/sdk/src/commands/init.ts`).
- **Emoji and variation selectors are bytes.** Copy emoji literals across exactly, including any trailing
  U+FE0F variation selector. Re-typing an emoji is how you lose (or gain) a code point and shift the bytes.
- **File modes are part of the output.** Written files are non-executable (`0644`); the parity harness
  compares mode. Don't let a port write `0755` or inherit a different umask-driven mode.
- **Directory order is `readdir().sort()`.** The written-files list and any directory walk follow sorted
  filename order. Renaming a file changes its sort position, which shifts where it appears in emitted
  lists; a rename is never just a rename for parity.

## Vendored assets are synced, never hand-edited

Each SDK carries copies of shared inputs. The canonical sources are repo-root `/templates`, `/schemas`,
and `packages/sdk/cli.json`; the per-SDK copies (including `packages/sdk/templates`, the SDK `_assets/`
dirs, and the Go/Py vendored `cli.json`) are generated by `npm run assets`. Edit the **canonical** source,
then run `npm run assets` to propagate and `npm run assets:check` to verify. The sync copies but **does not
prune**, so a renamed or deleted canonical file leaves a stale copy behind; remove the orphan by hand
after syncing.

## Versioning

The version lives in nine files kept coherent by `npm run version:set` (passing the new version); never
hand-edit them, and verify with `npm run version:check`. A published version is immutable; fixes go
forward in a new version.

## The gate (run before calling a port done)

- `npm run parity`: byte-identical across all three SDKs (the hard gate)
- `npm test --workspace packages/sdk`: Node reference suite
- `cd packages/sdk-go && go test ./... && gofmt -l .` (`gofmt -l` must print nothing)
- `cd packages/sdk-py && .venv/bin/python -m pytest -q`
- `npm run assets:check` and `npm run version:check`: vendored copies and version files in sync
